*関数詳細
	
**圧縮関数	
	
**解凍関数	
	
DecDCTReset	画像処理サブシステムの初期化

	形式	void DecDCTReset(int mode)
	
	引数	mode	リセットモード
			0: 全てを初期化
			1: 現在実行中の処理のみを中断

	解説	画像処理サブシステムをリセットする。
	
	返り値	なし

DecDCTvlc	VLC の解凍を行なう。

	形式	int DecDCTvlc(u_long *bs, u_long *buf)
	
	引数	bs	入力ビットストリーム
		buf	出力ランレベル

	解説	ビットストリーム bs を解凍し、中間形式であるランレベルを
		buf 以下に生成する。解凍後のランレベルの大きさは、
		DecDCTBufSize() で取得できる。
	
		ランレベルが DecDCTvlcSize() で指定された値を越えると、
		DecDCTvlc() は一旦処理を中断し制御をアプリケーションに
		戻す。中断した VLC デコードは、DecDCTvlc(0, 0) を実行
		することで再開できる

	返り値	0	全ビットストリームのデコードが終了
		1	未デコードのビットストリームを残して復帰
		-1	デコード失敗
		
	備考	ブロック関数。DecDCTin() を実行する前に必ず DecDCTvlc()
		で一旦ビットストリームをランレベルに変換しておく必要があ
		る。
		途中結果のランレベルを DecDCTin() に入力として与えるこ
		とはできない。
	
DecDCTvlcSize	一回の VLC の解凍の最大サイズを設定

	形式	int DecDCTvlcSize(int size)
	
	引数	size	デコードされたランレベルの最大値（ロングワード）
		
	解説	ビットストリーム bs を解凍する際の一回のデコードの最大
		値を与える。DecDCTvlc() は size ワードを越えて最初のブ
		ロックのランレベルをデコードするとデコードを中断する
		size に 0 を指定すると中断は発生しない。デフォルトの値
		は 0。
	
	備考	ブロック関数。DecDCTin() を実行する前に必ず DecDCTvlc()
		で一旦ビットストリームをランレベルに変換しておく必要があ
		る。
	
	返り値	直前に設定されたランレベルの最大値

DecDCTin	ランレベルを画像処理サブシステムに送信

	形式	void DecDCTin(u_long *buf, int mode)
	
	引数	buf	入力ランレベル
		mode	デコードモード
			    bit 0 出力モード
				  0	16bit ダイレクトカラー
				  1 	24bit ダイレクトカラー
			    bit 1 STP
				  0	STP=0
				  1	STP=1
	
	解説	ランレベル buf 以下をダイレクトカラー画像に解凍す
		る。解凍結果は、DecDCTout() で取り出す。
		出力のピクセルの深さは、24bit/16bit のどちらかを選択で
		きる。16bit モードの場合、解凍後のピクセルの 15bit目
		(STPビット）は、mode の bit1 で設定できる。
	
	返り値	なし

	備考	DecDCTin() で解凍された結果生じる画像の幅・高さに関す
		る情報は、ランレベルやビットストリームには保持されてい
		ない。これらのパラメータは、より上位の階層のデータ形式
		(STR フォーマットなど）で定義される。
	
		送受信のデッドロックを防止するために、送信したランレベル
		に対応するサイズのデータは DecDCTout() によって必ず受信
		しなくてはならない。
		しかし、その際に一回の DecDCTin() で送信したデータを複数
		回の DecDCTout() で読み出してもよいし、その逆もできる。

		DecDCTin() はノンブロック関数であるため、実際の転送の終
		了は DecDCTinSync() で検出する必要がある。DecDCTin() の
		終了をまたずに、次の DecDCTin() を実行すると２回目の送
		信は１回目の送信が終了するまでブロックされる。

DecDCTout	画像処理サブシステムからデコードデータを受けとる

	形式	void DecDCTout(u_long *buf, int size)
	
	引数	buf	解凍後画像データ
		size	受信するデータサイズ（ロングワード）

	解説	ランレベルを解凍し size ワード分の結果を buf 以下に格納
		する。解凍データのフォーマットは、DecDCTin() の引数に
		よって 24bit/16bit いずれかになる。
		サイズ size は全解凍データサイズと等しいか小さい値を指
		定する。全解凍データより小さい値を指定した場合は、送受信
		のデッドロックを防止するために、続くDecDCTout で残りのデ
		ータをすべて読み出す必要がある。
		解凍画像は、マクロブロック（16x16矩形領域）単位で出力
		される。データサイズはマクロブロックサイズ（16bit モー
		ドならば 16x16x2 = 512byte = 128 word, 24bit モードなら
		ば、16x16x3 = 768 byte = 192 word)単位で指定する必要が
		ある。
	
	返り値	なし

	備考	DecDCTout() はノンブロック関数であるため、実際の転送の
		終了は DecDCToutSync() で検出する必要がある。DecDCTout()
		の終了をまたずに、次の DecDCTout() を実行すると２回目の送
		信は１回目の送信が終了するまでブロックされる。

DecDCTinSync	画像処理サブシステムへの送信の終了を待つ

	形式	int DecDCTinSync(int mode)
	
	引数	mode	モード
			0: 終了までブロックする
			1: 状態の通知のみを行なう。

	解説	DecDCTin() 関数の終了を検出する。
			
	返り値	画像処理サブシステムの状態
			0: 送信が行なわれていない。	
			1: 送信中
	
DecDCToutSync	画像処理サブシステムへの受信の終了を待つ

	形式	int DecDCToutSync(int mode)
	
	引数	mode	モード
			0: 終了までブロックする
			1: 状態の通知のみを行なう。

	解説	DecDCTout() 関数の終了を検出する。
			
	返り値	画像処理サブシステムの状態
			0: 受信が行なわれていない。	
			1: 受信中
	
DecDCTinCallback	送信終了時に呼ばれるコールバックを設定

	形式	int DecDCTinCallback(void (*func)()) 
	
	引数	func	コールバック関数のアドレス

	解説	送信終了時に呼ばれるコールバック func を設定する。
		コールバックが設定されると、送信の終了時に関数 func が
		呼び出される。func に 0 を指定した場合は、コールバックは
		発生しない。

	返り値	なし
	
	備考	func 内では以降の送信終了割り込みはマスクされる。従って
		funcは必要な処理を終了したらすみやかにリターンする必要が
		ある。
	
DecDCToutCallback	受信終了時に呼ばれるコールバックを設定

	形式	int DecDCToutCallback(void (*func)()) 
	
	引数	func	コールバック関数のアドレス

	解説	受信終了時に呼ばれるコールバック func を設定する。
		コールバックが設定されると、受信の終了時に関数 func が
		呼び出される。func に 0 を指定した場合は、コールバックは
		発生しない。

	返り値	なし
	
	備考	func 内では以降の受信終了割り込みはマスクされる。従って
		funcは必要な処理を終了したらすみやかにリターンする必要が
		ある。
	
DecDCTBufSize	ランレベルのサイズを獲得する。

	形式	int DecDCTBufSize(u_long *bs)

	引数	bs	ビットストリーム

	解説	入力ビットストリーム bs を解凍し中間形式として生成され
		るランレベルのサイズを解凍せずに取得する。
		実際の解凍は行なわない。
	
	返り値	ランレベルのサイズ（ロングワード）
